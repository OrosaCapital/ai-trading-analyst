
LOVABLE_FIX_PACKAGE_FETCH_CHART_DATA_500
========================================

ERROR
Edge Function fetch-chart-data returned:
"Failed to fetch CoinGlass data for BTCUSD. Hobbyist plan requires valid 4h interval data."

CAUSE
- CoinGlass Hobbyist plan only supports interval=4h.
- fetch-chart-data is requesting unsupported intervals (1m/5m/1h).
- TradingView Lite is already the chart provider; backend chart fetch is unnecessary.

REQUIRED FIXES
--------------

1) REMOVE fetch-chart-data FROM FRONTEND
- Do NOT call /functions/fetch-chart-data for ANY symbol.
- TradingView Lite supplies all OHLCV candles.
- Delete all fetch-chart-data usage in UI.

2) UPDATE fetch-chart-data EDGE FUNCTION (if kept)
- Hardcode: interval="4h"
- Never request lower intervals.

3) SYMBOL NORMALIZATION
- Ensure all functions pass base symbols: BTC, ETH, SOL.
- Use symbolFormatter to prevent BTCUSD / BTCUSDT usage.

4) FRONTEND BLANK SCREEN FIX
- If backend errors, show message:
  "Chart data limited by provider. Displaying TradingView data."
- Do NOT display blank UI.

5) OPTIONAL DEPRECATION
- If TradingView handles all candles, keep fetch-chart-data disabled.

ENFORCEMENT RULES
-----------------
- Lovable must ensure:
  * All chart candles come from TradingView.
  * No Edge Function calls CoinGlass with invalid intervals.
  * Fallback to 4h if interval mismatch occurs.
  * No blank charts caused by backend failure.
